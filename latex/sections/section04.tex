\section{SECCIÓN 4. DOCUMENTACIÓN}

La Tabla 4-1 es una lista de los productos de software entregables del proyecto y el estándar o pautas asociados utilizados para desarrollar y mantener los productos de software. La Tabla 4-2 es una lista de productos no entregables.

\begin{enumerate}
    \item Todos los documentos se someterán a una revisión por pares de acuerdo con el Proceso de Revisión por Pares, referencia (n).
    \item El SQAP (CI-01) será revisado por todo el equipo y aprobado por el Gerente de Proyecto antes de la fase de planificación.
    \item El Plan de Pruebas (CI-04) será revisado antes de la ejecución de pruebas para verificar que los criterios de aceptación y la trazabilidad estén correctamente definidos.
    \item El Informe de Pruebas (CI-05) será revisado tras la ejecución de pruebas, registrando hallazgos en el Formulario de Reporte de Discrepancias (DCR) del Apéndice.
    \item La documentación de usuario (CI-06) y de despliegue (CI-07) serán verificadas mediante revisión por pares antes de su liberación.
    \item Los productos no entregables (código fuente, configuraciones, scripts) serán verificados continuamente mediante revisiones de código, análisis estático (SonarQube, Qlty) y herramientas de cobertura (JaCoCo).
\end{enumerate}

Tras la finalización de una revisión por pares, SQA registra y reporta las mediciones de la revisión por pares (el elemento revisado, el número de errores detectados, la fase en la que se realizó la revisión por pares, el número de informes de error cerrados y el número de informes de error abiertos) a SEPO.

Tras la finalización de una revisión por pares, el producto de software se enviará a SCM y se colocará bajo control de CM. El producto de software se procesará entonces de acuerdo con el proceso de aprobación y liberación de productos de software de SCM descrito en la referencia (e).
